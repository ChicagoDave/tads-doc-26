## Key Decisions

### 1. Grouped Language Reference into 7 Subsections
The language section contains 40+ individual pages. Rather than listing them all flat, the landing page groups them into 7 logical subsections (Core Language, Types and Values, Object Model, Procedural Code, Advanced Features, System Library, Web UI) to give readers a navigable overview. Each subsection bullet links to the first or most representative page in the group.

### 2. Listed Every Intrinsic Class Individually
Unlike the language section, the intrinsics section's 27 classes are each a distinct, self-contained reference page with no further hierarchy. Listing them individually gives the landing page genuine reference value.

### 3. Organized Library Landing by Function Rather Than Nav Order
The adv3 library guide nav ordering is not strictly logical from a reader's perspective. The library landing page groups content into functional categories (game design, core topics, output and formatting, extras) to help readers find what they need faster.

### 4. VM Spec Organized into Three Tiers
The VM spec subsections (overview, architecture, reference) map naturally to depth of engagement: conceptual introduction, structural description, and detailed reference. The landing page makes this progression explicit.
